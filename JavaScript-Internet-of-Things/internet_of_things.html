What is Internet of Things or IoT?

Internet of Things or IoT ဆိုတာ electronics, sensors နဲ့ software တို့ကို internet နဲ့ချိတ်ဆက်ထားတဲ့ physical objects တွေအဖြစ် ဖော်ပြထားပါတယ်။ Data ကို collected လုပ်တယ် (သို့) devices တွေဆီကို sent လုပ်ပေးတာဖြစ်ပါတယ်။ Internet of Things devices တွေဟာ internet နဲ့ connect လုပ်တဲ့အခါမှာ နည်းလမ်းအမျိုးမျိုးရှိပါတယ်။ Router ဆီကို via WiFi or Ethernet နဲ့ တိုက်ရိုက် connect လုပ်ပေးတယ် (သို့မဟုတ်) IoT devices တွေဟာ smartphone မှာပါတဲ့ via Bluetooth ကဲ့သို့သော geteway device တစ်ခုနဲ့ connect လုပ်နိုင်ပါတယ်။ Zigbee နဲ့ Z-Wave လိုမျိုး အခြား wireless technologies တွေလည်း ရှိပါသေးတယ်။ အဲ့ဒီ devices တွေကတော့ hubs ဆိုတာနဲ့ connect လုပ်ပြီးမှ phone or Internet services တွေကို ပြန်ပြီး connect လုပ်ပေးတာပါ။ ဥပမာ Samsung မှာဆိုရင် Internet နဲ့ ချိတ်ဆက်ပေးတဲ့ security cameras တွေကို produce လုပ်ပေးပါတယ်။ Big data နဲ့ IoT ရဲ့ combination ကြောင့် လယ်သမားတွေဟာ သီးနှံတွေအထွက်တိုးဖို့ algorithms တွေအသုံးပြုတာလိုမျိုး၊ Automatic industry တွေဟာ employee တွေ safe ဖြစ်စေဖို့ accident တစ်ခုဖြစ်ခဲ့ရင် emergency services တွေ alerting လုပ်ပြတာမျိုး၊ consumer electric meterတွေကကျွန်တော်တို့energy consumption ကိုကူညီပေးတယ်တာမျိုး ၊ပြီးတော့electricity နဲ့ ပိုက်ဆံကိုဘယ်လို save လုပ်ရမယ်ဆိုတဲ့ suggestion တွေကိုပေးပါတယ်။

Single Board Computers

ကျွန်တော်တို့အနေနဲ့ Internet of Things အတွက် JavaScript ကို ဘာကြောင့်အသုံးပြုရလဲလို့ မေးကောင်းမေးပါလိမ့်မယ်။ ဘာကြောင့်အသုံးပြုသင့်လဲဆိုရင်တော့ ပထမဆုံး သူဟာ asynchronous & event-driven ဖြစ်လို့ပဲ ဖြစ်ပါတယ်။ ကျွန်တော်တို့က input or sensor data ကို read လုပ်လိုက်တဲ့အချိန် event တစ်ခုတွေ့မယ်ဆိုရင် အဲ့ဒီ input or sensor data ကို EventListener တစ်ခုကို trigger လုပ်လို့ရအောင် add လုပ်ပေးရပါတယ်။ ဒါဟာ DOM element ထဲမှာ click listener ထည့်လိုက်တာနဲ့ တူတူပါပဲ။ အခြား languages တွေမှာလည်း သူတို့ရဲ့ asynchronous programming ဟာ JavaScript ရဲ့ built-in အဖြစ်ရှိကြပါတယ်။ JavaScript ကို browser ပေါ်က dashboard ကနေ server side က data တွေကို collect လုပ်ဖို့ နဲ့ electronics တွေမှာလည်း နေရာတိုင်းလိုလိုအသုံးပြုနိုင်ပါတယ်။ JavaScript ဟာ ကျွန်တော်တို့ကို architect solutions နဲ့ one language မှာ ပြီးမြောက်စေဖို့ allow လုပ်ပေးပါတယ်။ JavaScript နဲ့ Internet of Things မှာ strategy 3ခု ရှိပါတယ်။ ပထမတစ်ခုက Single Board Computes ၊ ဒုတိယတစ်ခုက JavaScript Micro-controller နဲ့ တတိယတစ်ခုကတော့ Client-Host Model တို့ပဲဖြစ်ပါတယ်။ Single Board Computers ဆိုတာကတော့ processing power, RAM နဲ့ single board တစ်ခုမှာရှိတဲ့ storage space တွေအားလုံးရှိတဲ့ Computers တွေပဲဖြစ်ပါတယ်။ ဘာလို့လဲဆိုရင်တော့ သူတို့ဟာ full computers တွေဖြစ်ပြီး node.js ကိုပါ လုပ်ဆောင်ပေးနိုင်ပါတယ်။ Single board computers တွေကတစ်ခုကတော့ Raspberry Pi ဖြစ်ပြီး သူ့မှာ Wi-Fi, internet နဲ့ Bluetooth တို့ built-in အနေနဲ့ ပါရှိပါတယ်။ Board ရဲ့ edge မှာလည်း inputs နဲ့ outputs တွေအတွက် အသုံးပြုနိုင်တဲ့ pins 40 လောက်ရှိပါတယ်။

Microcontrollers

Microcontrollers တွေဟာ processing power နဲ့ RAM အနည်းငယ်သာပါဝင်တဲ့ instant-on computers တွေနဲ့ specialize လုပ်ထားတာဖြစ်ပါတယ်။ ဒါပေမယ့် သူတို့မှာ မယုံနိုင်လောက်အောင် မြန်ဆန်တဲ့ IEO ရှိပါတယ်။ ဘာလို့လဲဆိုရင် သူတို့ကို power လုံလုံလောက်လောက်ရရှိပြီးတော့ single uses တွေအတွက် specialize လုပ်ထားလို့ပဲဖြစ်ပါတယ်။ Processing power နည်းခြင်းဟာ interpreted language ကနေ running လုပ်ခြင်းကို prevent လုပ်မပေးပါဘူး။ Espruino, Smart.js, XS6 နဲ့ Sausungရဲ့ Jerry Script တွေကဲ့သို့သော microcollers ပေါ်မှာ run နိုင်တဲ့ specialized လုပ်ထားတဲ့ JavaScript Runtimes တွေ အများကြီးရှိပါတယ်။ Espruino Runtime ကို အခြား microcontrollers တွေနဲ့ အလုပ်လုပ်ဖို့အတွက် ported လုပ်ထားပါတယ်။ ESP8266 ဆိုတဲ့ microcontroller ကတော့ WiFi ပါဝင်ပြီးတော့ $2 ပဲကုန်ကျပါမယ်။ Feather HUZZAH မှာတော့ သူ့ထဲမှာ embed လုပ်ထားတဲ့ ESP8266 တစ်ခုရှိပါတယ်။ ESP8266 က အဲ့ဒီအပေါ်မှာဆိုရင် JavaScript မရှိပါဘူး။ ဒါကြောင့် device ပေါ်မှာ install လုပ်ရပါတယ်။ အဲ့ဒီ installation process ကိုတော့ Flashing လို့ခေါ်ပါတယ်။ ကျွန်တောတို့အနေနဲ့ device ပေါ်မှာ JavaScript Runtime ကို install လုပ်ဖို့ Python based tool, ESP tool or graphical user interface, Flasher.js ကို အသုံးပြုနိုင်ပါတယ်။ ES6 နဲ့လာတဲ့ Kinoma Element ကို သူ့ထဲမှာ built လုပ်ထားတာဖြစ်တဲ့အတွက် Flashing မလိုအပ်တော့ပါဘူး။

Client-Host Model

Internet of Things မှာရှိတဲ့ JavaScript အတွက် strategies 3ခုထဲက နောက်ဆုံးတစ်ခုကတော့ Client-Host Model ပဲဖြစ်ပါတယ်။ ဒါကတော့ ကျွန်တော်တို့ microcontroller တစ်ခုပေါ်က software တွေကို ပိုပြီး powerful ဖြစ်တဲ့ computer ဆီကနေ instructions တွေကိုစောင့်ပြီးတော့ install လုပ်တာပဲဖြစ်ပါတယ်။ ဒီလိုပြုလုပ်ဖို့အတွက် ကျွန်တော်တို့မှာ device ဆီကို flashed လုပ်ထားတဲ့ specialized software နဲ့ microcontroller တစ်ခုလိုအပ်ပါတယ်။ အများဆုံးအသုံးပြုတဲ့ microcontroller ကတော့ Firmata ပဲဖြစ်ပါတယ်။ ဒုတိယတစ်ခုအနေနဲ့ Firmata protocol ကိုသုံးပြီး microcontroller ကို ဘာလုပ်ရမလဲဆိုတာပြောဖို့အတွက် Node.js running လုပ်နေတဲ့ host computer လိုအပ်ပါလိမ့်မယ်။ Common ဖြစ်တဲ့ microcontroller bolts တွေထဲကတစ်ခုကတော့ Arduino Uno ဖြစ်ပါတယ်။ Computers တွေက UNO, A USB တို့နဲ့ communicate လုပ်ပါတယ်။ Arduino 101 ဟာဆိုရင် wireless ချိတ်လို့ရတဲ့ Intel Bluetooth chip ပါရှိပါတယ်။ Firmata ကို ESP8266 ဆီ flash လုပ်နိုင်ပြီးတော့ Wi-Fi ပေါ်မှာ talk လုပ်နိုင်ပါတယ်။ Node.js မှာ running လုပ်နေတဲ့ host computer ဟာ laptop or desktop တစ်ခုရှိဖို့မလိုပါဘူး။ သူဟာ Raspberry Pi လိုမျိုး single board computer ဖြစ်နိုင်ပါတယ်။ Node.js ကို addition အနေနဲ့ပြောရရင် Firmata ပြောတတ်တဲ့ framework တစ်ခုလိုအပ်ပါတယ်။ Johnny-Five နဲ့ Cylon.js libraries တွေကတော့ Firmata ပြောတတ်ပါတယ်။

Defining Variables with let and const

Declaring Variables in JavaScript

JavaScript ကိုစလေ့လာပြီဆိုတာနဲ့ အမြဲတမ်းတွေ့ရမှာကတော့ variables တွေပါပဲ။ ကျွန်တော်တို့ဟာ variable တစ်ခုကို create or declare လုပ်တဲ့ နည်းလမ်းတစ်ခုရှိပါတယ်။ အဲ့ဒါကတော့ var ဆိုတဲ့ keyword ဖြစ်ပြီး JavaScript programmers တော်တော်များများကတော့ အဲ့ဒီ method ကိုအသုံးပြုကြပါတယ်။ ဒါပေမယ့် တစ်ချို့သော behaviors တွေမှာ var ကိုသုံးခြင်းက problematic or unexpected ဖြစ်နိုင်ပါတယ်။ ဒါကြောင့် JavaScript programming language မှာရှိတဲ့ developers တွေဟာ variables တွေကို create လုပ်ဖို့ နည်းလမ်းအသစ်တွေကို မိတ်ဆက်ပေးဖို့ ဆုံးဖြတ်ခဲ့ကြပါတယ်။ ES2015 လို့ခေါ်တဲ့ JavaScript 6th version (သို့) ECMAScript 2015 မှာ const နဲ့ let ဆိုတဲ့ new keyword နှစ်ခုကို မိတ်ဆက်လိုက်ပါတယ်။ အခု const နဲ့ let က ဘယ်လိုကွာခြားပြီး ဘယ်လို အလုပ်လုပ်သလဲဆိုတာ လေ့လာကြည့်ရအောင်။ ပြီးတော့ var နဲ့ကြေငြာမယ့်အစား ဘာကြောင့် const နဲ့ let ကို သုံးရသလဲဆိုတာကိုရောပါ လေ့လာပါမယ်။ ဒါပေမယ့် ပထမဆုံးပြောချင်တာကတော့ const နဲ့ let ဟာ var လိုမျိုးပဲ တူညီတဲ့နည်းလမ်းတွေပဲဖြစ်ပါတယ်။ var or const or let လို့အရင်ဆုံးရေးပြီး သူ့နောက်မှာ variable name ရေးပေးရပါတယ်။ ပြီးတော့မှာ အဲ့ဒီ variable ထဲကို equal sign အသုံးပြုပြီး ထည့်ချင်တဲ့ value ကို ထည့်ပေးရတာပါ။

For Example:

        
            var message = "Hello!";
            const pi = 3.14159;
            let score = 0;

Creating Unchanging Variables with const

ကျွန်တော်တို့အနေနဲ့ variable တစ်ခုကို create လုပ်မယ့်ဆိုရင် အရင်ဆုံးစဉ်းစားသင့်တာက variable declare လုပ်နည်း သုံးမျိုးထဲက const ဆိုတဲ့ keyword ပဲဖြစ်ပါတယ်။ ဒါဟာ declaring variables လုပ်တာအတွက် best choice တစ်ခုဖြစ်ပါတယ်။ Const ဆိုတာ constant လို့ပြောတာဖြစ်ပြီး ဆိုလိုတာကတော့ အဲ့ဒီ const နဲ့ကြေငြာလိုက်တဲ့ variable ထဲက value ဟာ အသေအနေနဲ့ရှိနေသင့်ပြီး လုံးဝမပြောင်းလဲသင့်ပါဘူး။ ဥပမာ const pi = 3.14159 လိုမျိုး pi တန်ဖို့ဟာ လုံးဝပြောင်းလဲပစ်လို့မရပါဘူး။ Const ဟာ variables တွေထဲကို ကျွန်တော်တို့ code ထဲက ဘယ် assign လုပ်မယ့် value တွေဖြစ်ပါစေ overwrite လာလုပ်တာကို ကာကွယ်ပေးပါတယ်။ တစ်နည်းအားဖြင့် constant တစ်ခုကြေငြာလိုက်တာနဲ့ သူ့ကို အခြား value တစ်ခုထပ်ပေးလို့ မရတော့ပါဘူး။ ကျွန်တော်တို့အနေနဲ့ const pi = 3.14159 ဆိုတဲ့ constant variable ထဲကို အခြား value တစ်ခုထပ်ထည့်မယ်ဆိုရင်တော့ Uncaught TypeError: Assignment to a constant variable ဆိုတဲ့ error ကို ပြပေးမှာဖြစ်ပါတယ်။ ဒါကြောင့် ကျွန်တော်တို့က value အသေဖြစ်ချင်တဲ့ variable တစ်ခုကို ကြေငြာမယ်ဆိုရင် const ကိုသုံးသင့်ပါတယ်။ ဥပမာ score လိုမျိုး game ရဲ့ score တွေက အမြဲတမ်းပြောင်းနေမှာဖြစ်တဲ့ အခြေအနေမျိုးမှာဆိုရင်တော့ let နဲ့ ကြေငြာသင့်ပါတယ်။

For Example:

            </html>

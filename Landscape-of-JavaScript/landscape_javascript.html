The Landscape of JavaScript

ယနေ့ခေတ်မှာ JavaScript ဟာဆိုရင် သူ့ရဲ့ power နဲ့ ability တွေကြောင့် modern developer အတွက် မရှိမဖြစ် လေ့လာသင့်တဲ့ skills တွေထဲက တစ်ခုဖြစ်လာပါတယ်။ ဒီတစ်ခါမှာတော့ ကျွန်တော်တို့အနေနဲ့ JavaScript ရဲ့ modern landscape ကို လေ့လာသွားမှာဖြစ်ပါတယ်။ JavaScript ဘယ်လိုပေါ်ပေါက်လာသလဲဆိုတာကို နားလည်ထားခြင်းက အခု current state မှာရှိတဲ့ JavaScript ကို ပိုပြီးအထောက်အကူပြုနိုင်ပါလိမ့်မယ်။ JavaScript စပေါ်လာတဲ့အချိန်တည်းက Browser နဲ့ စတင်လာခဲ့တာဖြစ်ပါတယ်။ 1995, May လမှာ Internet နဲ့ Web ဟာ အပြိုင်ပေါ်လာပြီး အဲ့ဒီအချိန်တုန်းက Web ဟာ static အနေနဲ့ပဲ ရှိပါသေးတယ်။ Water skiing puppies ရဲ့ videos ၊ Juggling cats ရဲ့ photo gallery တွေကို web ပေါ်မှာ အမြဲတမ်းကြည့်လို့မရသေးပါဘူး။ ကျွန်တော်တို့ဟာ web pages တွေကို program မလုပ်နိုင်သေးတာကြောင့် အဲ့ဒီအချိန်တုန်းက text တွေဟာ heavy ဖြစ်နေပြီး text နဲ့ image တွေဟာလည်း left side နဲ့ right side မှာပဲ float ဖြစ်နေပါတယ်။ အဲ့ဒီအချိန်မှာ Netscape Communications ဆိုတဲ့ company တစ်ခုက ပထမဆုံး Netscape Navigator ဆိုတဲ့ commercial web browser ကို ဖန်တီးလိုက်ပါတယ်။ အဲ့ဒီ browser ကအောင်မြင်ပြီး မကြာခင်မှာ Netscape ရဲ့ founder ဖြစ်တဲ့ Marc Andressen ဟာ web ကို ပိုပြီးတော့ livelier ဖြစ်ဖို့နဲ့ dynamic ကျကျ လုပ်ဆောင်ဖို့ လိုအပ်နေသေးတယ်လို့ တွေးမိပါတယ်။ သူက HTML ဟာ scripting language တစ်ခုလိုအပ်နေတယ်လို့ ယုံကြည်ပြီး browser က native ကျကျ run နိုင်မယ်၊ ပြီးတော့ website ကို functionality အချို့ add လုပ်ဖို့လိုလိမ့်မယ်လို့ တွေးမိလိုက်ပါတယ်။ ဒါကြောင့် သူဟာ Scheme ဆိုတဲ့ type မတူတဲ့ programming language တစ်ခုကို Netscape navigator ထဲကို embed လုပ်ဖို့အတွက် ငှားထားတဲ့ engineer တစ်ယောက်ဖြစ်တဲ့ Brendan Eich ကို ပေးအပ်ခဲ့ပါတယ်။ ဒါပေမယ့် သူဟာ project ကိုမစခင်မှာပဲ Eich ဟာ new scripting language ကို စတင်ပြီးအလုပ်လုပ်ဖို့အတွက် pull လုပ်နေပါပြီ။ အဲ့ဒါနဲ့ပဲ 10 ရက်လောက်အကြာမှာ Eich ဟာ new programming language တစ်ခုဖြစ်တဲ့ Mocha ကို design လုပ်ခဲ့ပါတယ်။ သူ့ကိုအစောပိုင်းက JavaScript လို့ name ပေးခဲ့တာပါ။ အဲ့ဒီလိုနဲ့ပဲ language ဟာ LiveScript လို့ပြောင်းသွားပြီး သူ့ကို Netscape Navigator ရဲ့ version 2.0 မှာ တင်ပေးလိုက်ပါတယ်။ အဲ့ဒီမှာ Netscape က Java language မှာအောင်မြင်နေတဲ့အချိန်ဖြစ်တာကြောင့် name ကို JavaScript အဖြစ် officially အရကြေငြာခဲ့ပါတယ်။ ဒါပေမယ့် Java နဲ့ JavaScript ကတော့ ဆက်စပ်မှုမရှိပါဘူး။ အဲ့ကနေစပြီး ယနေ့မှာ JavaScript ဟာ web ရဲ့ programming language တစ်ခုဖြစ်နေပါပြီ။ သူ့ကို major browser အားလုံးမှာ run လို့ရပါတယ်။ နောက်သင်ခန်းစာတွေမှာ JavaScript ရဲ့ evolution နဲ့ developer တွေရနိုင်တဲ့ JavaScript ရဲ့ flavor types တွေကို လေ့လာသွားပါမယ်။

How Useful JavaScript?

JavaScript ဟာ အရင်တုန်းက မူလ browser မှာပဲအလုပ်လုပ်နိုင်အောင် design လုပ်ထားတယ်ဆိုပေမယ့် အခုဆိုရင် multi platform language တစ်ခုပေါ်မှာပါ အလုပ်လုပ်နိုင်နေပါပြီ။ သူ့ကို native application development, virtual reality, build chat, voice controlled bots, games နဲ့ အခြား ကျွန်တော်တို့ think လုပ်နိုင်ရင် လုပ်နိုင်သလောက် application တွေမှာ အသုံးပြုနိုင်ပြီဖြစ်ပါတယ်။ JavaScript ကို encounter လုပ်ဖို့အတွက် most common place တစ်ခုကတော့ web ပေါ်မှာပဲဖြစ်ပါတယ်။ Developers တွေဟာ websites တွေကို ပိုပြီး ဆွဲဆောင်မှုရှိဖို့နဲ့ ပျော်စရာကောင်းဖို့အတွက် JavaScript ကိုအသုံးပြုပါတယ်။ 3D graphics နဲ့ smooth animations လိုမျိုး user ဆီပေးနိုင်တဲ့ interactions အားလုံးကို JavaScript နဲ့ပြုလုပ်နိုင်ပါတယ်။ JavaScript ဟာ Google Apps နဲ့ Microsoft Office လိုမျိုး popular productivity apps တွေအတွက် add-ons တွေလည်း create လုပ်နိုင်ပါတယ်။ ဒါတင်မကဘဲ Android နဲ့ iOS တွေမှာ run နိုင်တဲ့ native apps တွေမှာပါ JavaScript ကိုအသုံးပြုနိုင်ပါတယ်။ ကျွန်တော်တို့ဟာ HTML, CSS နဲ့ JavaScript only ကိုအသုံးပြုပြီး cross-platform desktop applications တွေလည်း ဖန်တီးနိုင်ပါတယ်။ ဥပမာ Visual Studio Code ဆိုတဲ့ text editor နဲ့ Slack ဆိုတဲ့ messaging tool ဟာ JavaScript ကိုသုံးပြီးရေးထားတာဖြစ်ပါတယ်။ ပြီးတော့ developers တွေဟာ web ပေါ်က virtual reality experiences တွေ create လုပ်ဖို့အတွက်လည်း JavaScript ကိုအသုံးပြုပါတယ်။ JavaScript ဟာ hardware ပေါ်မှာတောင် run နိုင်ပါတယ်။ သူဟာ Internet of Things or IoT devices တွေ ပြုလုပ်ပေးတဲ့ micro-controllers တွေမှာ running လုပ်နိုင်ပါတယ်။ IoT device ဆိုတာ Internet or Iot devices နဲ့ connect လုပ်နိုင်တဲ့ on & off ခလုပ်ပါတဲ့ ဘယ် device မဆို ဆိုလိုပါတယ်။ WiFi connected coffee maker, headphones, နဲ့ washing machine တွေဟာလည်း JavaScript ကိုအသုံးပြုထားတာဖြစ်ပါတယ်။ JavaScript ဟာ ဒီလိုနဲ့ပဲ Machine Learning လုပ်တဲ့နည်းလမ်းကို တွေ့ခဲ့ပါတယ်။ ဒါကြောင့် သင့်အနေနဲ့ machine learning ကို လေ့လာချင်တဲ့ JavaScript developer ဖြစ်တယ်ဆိုရင်တော့ TensorFlow.js ဟာ machine learning ကိုလေ့လာဖို့ အကောင်းဆုံးနေရာဖြစ်ပါတယ်။ ပြီးတော့ NASA ဟာလည်း open source JavaScript environment တစ်ခုဖြစ်တဲ့ Node.js ကို သူတို့ရဲ့ space suits တွေပြင်ဆင်ဖို့နဲ့ space ထဲမှာ အာကာသယာဉ်မှူးတွေ safe ဖြစ်ဖို့အတွက် အသုံးပြုနေကြပါတယ်။ ဒါကြောင့် JavaScript ဟာ browser based language တစ်ခုဖြစ်တာကြောင့် နေရာတော်တော်များများမှာ တွေ့နေရတာဖြစ်ပါတယ်။

Involved JavaScript

JavaScript က powerful robust programming language တစ်ခုအနေနဲ့ ရပ်တည်လာတာ နှစ်ပေါင်းအတော်ကြာခဲ့ပီဖြစ်ပါတယ်။ Netscape Navigator မှာjavaScript က officially ထွက်တဲ့အချိန်မှာ game changer တွေဖြစ်လာပါတယ်။ ဒါကြောင့် browser တွေကမကူညီနိုင်တော့ပါဘူး။ JavaScript ကသူရဲ့own version ထွက်လာပါတယ်။ Microsoft ကသူတို့ browser, Internet Explorer အတွက် scripting language တစ်ခုကိုလိုအပ်ခဲ့ပါတယ်။ ဒါကြောင်သူတို့က JavaScript ရဲ့own implementation ကို create လုပ်ခဲ့ပါတယ်။ သူတို့version ကို JScript လို့နာမည်ပေးခဲ့ပါတယ်။ JavaScript နဲ့ JScript ရဲ့differences ကတော့ designers နဲ့ developer တွေအတွက် website တစ်ခုကို browser နှစ်ခုမှာအလုပ်လုပ်ဖို့ခက်ခဲတာဖြစ်ပါတယ်။ Siteတစ်ခုက Netscape မှာအလုပ်လုပ်တယ်။ Internet Explorerနဲ့ အလုပ်မလုပ်ဘူး။ internet explorer မှာလုပ်ပီး Netscape မှာအလုပ်မလုပ်ဘူးစတာတွေဖြစ်ပါတယ်။ ဒါကြောင့် site တစ်ခုကို developing လုပ်တဲ့အခါ Netscape နဲ့ IE တွေနှစ်ခုလုံးမှာအလုပ်လုပ်ဖို့လိုအပ်ပါတယ်။ JavaScript က block တွေကို web ထဲ forward လုပ်တာတွေနာမည်ကြီးပါတယ်။ Forward လုပ်တဲ့အချိန်မှာ JavaScript က standardized မဖြစ်ပါဘူး။ Rule တွေမရှိဘူး။ ပြီးတော့ browser မှာ implementing လုပ်ဖို့guidelines တွေရှိပါတယ်။ 1996 မှာ Netscapeက JavaScriptကို standardလုပ်ဖို့ဆုံးဖြတ်ခဲ့ပါတယ်။ သူတို့က JavaScript ကို standardize လုပ်ဖို့ organization ECMA ကို submitted လုပ်ခဲ့ပါတယ်။ ECMAScript က official nameဖြစ်လာပြီး version သတ်မှတ်ဖို့စဉ်းစားလာကြပါတယ်။ Language က TC39 လို့ခေါ်တဲ့ ECMAမှာ technical တစ်ယောက်အောက်မှာ standardizedဖြစ်လာခဲ့ပါတယ်။ ECMAScript ရဲ့first edition ကို 1997 မှာထွက်ရှိခဲ့ပါတယ်။ ECMAScript 2 and 3 ကိုတော့နောက်ပိုင်းမှာ feature အသစ်တွေအနေနဲ့မိတ်ဆက်လိုက်ပါတယ်။ ဒါပေမယ့် evolving နဲ့ growing ရဲ့ process တွေက challenging တစ်ခုဖြစ်လာပါတယ်။ ECMAScript 4 မှာအလုပ်လုပ်တဲ့ TC39 ကတော့ အဲ့ဒီ featureတွေကိုသဘောမတူပါဘူး။ 2003မှာတော့ JavaScript က move forwardလုပ်လိုက်ပါတယ်။Committee ကဆက်ပြီးမလုပ်တော့ပဲ TC39 လည်းbroke upဖြစ်သွားပြီး spec တွေလည်း abandonedဖြစ်သွားခဲ့ပါတယ်။ ဒါပေမယ့် 2008 နောက်ပိုင်းမှာတော့ TC39 committee ကတော့ ECMAScript 1.1 လုပ်ဖို့group ကိုပြန်လည်ဖွဲ့စည်းခဲ့ပါတယ်။ အဲ့ဒါကိုတော့ ECMAScript 5လို့နာမည်ပြောင်းခဲ့ပါတယ်။ ES 5 မှာဆိုရင် JSON, getter, setter နဲ့ array method အသစ်တွေဖြစ်တဲ့ foreach and map တွေကိုမိတ်ဆက်ခဲ့ပါတယ်။ ES 5 က JavaScript ရဲ့version တစ်ခုဖြစ်ပါတယ်။ပြီးတော့ modern browser တွေမှာအပြည့်အ၀ support ပေးပါတယ်။ EM 4 က abandoned ဖြစ်ခဲ့သော်လည်း feature တချို့ကတော့ sixth edition ဖြစ်တဲ့ ES 6 မှာဆက်လက်ပါလာပါတယ်။ ES 6ကတော့ နှစ်ပေါင်းများစွာအသုံးပြုခဲ့ကြပြီး JavaScript history ရဲ့အကြီးမားဆုံးအစိတ်ပိုင်းတစ်ခုဖြစ်ပါတယ်။ နောက်ဆုံးထွက်တာကတော့ June 2015 က ECMAScript 2015 or ES 2015 ဆိုပြီး လူသိများလာပါတယ်။ အခုဆိုရင် TC39 Committee က Netflix, Facebook နဲ့ Paypal မှာဆက်လက်လုပ်ဆောင်နေပါတယ်။

JavaScript Flavors or Types

အခုသင်ခန်းစာမှာ ကျွန်တော်တို့က developers တွေရနိုင်တဲ့ JavaScript ရဲ့ flavors or types တွေအကြောင်းကို လေ့လာသွားမှာဖြစ်ပါတယ်။ ကျွန်တော်တို့အနေနဲ့ terms တွေဖြစ်တဲ့ Vanilla JavaScript, Plain or Pure JavaScript တွေကို ကြားဖူးပါလိမ့်မယ်။ သူတို့အားလုံးဟာ ဘယ် libraries တွေနဲ့ ဘယ် frameworks တွေကိုမှမသုံးဘဲ JavaScript ကိုပဲ အသုံးပြုထားတာဖြစ်ပါတယ်။ ဒါ့ဟာ developers တွေကို native ဖြစ်ဖို့နဲ့

positioning တွေကဲ့သို့သော CSS lab methods တွေမှာ အချို့သော limitations တွေရှိပြီး ယနေ့ complex responsive websites တွေရဲ့ layout demands ကို handle မလုပ်နိုင်ပါဘူး။ ကျွန်တော်တို့ CSS Layout Basics မှာလေ့လာခဲ့တဲ့အတိုင်း floats တွေဟာ သူတို့ရဲ့ quirky behavior ကြောင့် problematic ဖြစ်နိုင်ပါတယ်။ သူတို့ဟာ collapsing မှ elements တွေပါဝင်နေတာကို prevent လုပ်ဖို့အတွက် clear fixes တွေလိုအပ်ပါတယ်။ ပြီးတော့ inline-block display နဲ့ဆိုရင် ကျွန်တော်တို့ဟာ browser ကနေ create လိုက်တဲ့ default white space နဲ့ fight လုပ်ရပါမယ်။ ပြီးတော့ positioning ဟာလည်း page layout မှာ သတိထားရပါမယ်။ ဘာလို့လဲဆိုရင်တော့ သူဟာ context ကို position ချဖို့နဲ့ positioned ချထားတဲ့ elements တွေအမှန်တစ်ကယ် မရှိဘူးဆိုရင်တောင် page ပေါ်က အခြား content တွေကို position ချပေးဖို့လိုအပ်လို့ပဲဖြစ်ပါတယ်။ ဒါကြောင့်မို့လို့ Flexbox or Flexible Box Layout ဟာ page content ကို layout ချဖို့အတွက် flexible way တစ်ခုဖြစ်တဲ့ set of CSS properties ပဲဖြစ်ပါတယ်။ သူဟာ remarkable layout feature တစ်ခုဖြစ်ပြီး web designers တွေကို layouts တွေဘယ်လို build လုပ်ရမလဲဆိုတာကို ပြပေးပါတယ်။ ကျွန်တော်တို့ဟာ flexbox ကိုသုံးခြင်းအားဖြင့် stacking elements နဲ့ သူတို့ကိုသုံးပြီး အခြား strange ဖြစ်တဲ့ browser behaviors တွေမရှိဘဲ elements တွေ side by side ဖြစ်အောင် layout ချနိုင်တဲ့ floats, block နဲ့ inline-block တွေရဲ့ benefitsတွေကို ရရှိနိုင်ပါတယ်။ Flexbox ကိုသုံးခြင်းအားဖြင့် ကျွန်တော်တို့ဟာ navigation links, The Concepts of Flexbox

Flexbox ကိုအသုံးပြုပြီး layout မတည်ဆောက်ခင်မှာ ကျွန်တော်တို့အနေနဲ့ Flexbox ရဲ့ concepts နဲ့ terminology တွေကို နားလည်ထားသင့်ပါတယ်။ Flexbox layout မှာရှိတဲ့ အရေးကြီးတဲ့ elements နှစ်ခုကတော့ Flex containers နဲ့ Flex items တို့ပဲဖြစ်ပါတယ်။ ကျွန်တော်တို့ layout မှာ ပထမဆုံး flex ကိုသုံးမယ်ဆိုရင် အရင်ဆုံး Flex Container ကိုတည်ဆောက်ရပါတယ်။ Flex container ဟာ Flexbox layout အတွက် context ကို set လုပ်ပေးပြီး ၊ ကျွန်တော်တို့ Flexbox ကိုသုံးပြီး အမှန်တစ်ကယ် layout ချလိုက်တဲ့ elements တွေဖြစ်တဲ့ Flex items တွေပါဝင်ပါတယ်။ Flex container ဟာ div, ul, header နဲ့ apan element တွေလိုမျိုး ဘယ် block-level or inline element မဆို ဖြစ်နိုင်ပါတယ်။ Flex container တစ်ခုရဲ့ ဘယ် direct child ကိုမဆို Flex item လို့ခေါ်ပြီး အဲ့ဒီ flex container ထဲမှာ flex items တွေအများကြီးပါဝင်နိုင်ပါတယ်။ ဥပမာပြောရမယ်ဆိုရင် unordered list တစ်ခုမှာဆိုရင် ul ဟာ flex container ဖြစ်ပြီး li ဟာ flex items တွေပဲဖြစ်ပါတယ်။ Flex container နဲ့ flex items တွေ ဘယ်လိုအလုပ်လုပ်လဲဆိုတာ သိဖို့ဆိုရင် ကျွန်တော်တို့အနေနဲ့ browser တစ်ခုက flex items တွေ place လုပ်နိုင်တဲ့ axis အကြောင်းကို သိထားဖို့လိုအပ်ပါတယ်။ Flexbox ဟာ Flex items တွေရဲ့ layout direction ကို determine လုပ်ပေးတဲ့ axis နှစ်ခုရှိပါတယ်။ Container တစ်ခုမှာရှိတဲ့ items တွေဟာ main axis or cross axis မှာ laid out ဖြစ်နေကြပါတယ်။ ဒါကြောင့် Flexbox layout တစ်ခုအတွင်းမှာရှိတဲ့အရာအားလုံးဟာ အဲ့ဒီ axis နှစ်ခုမှာ relative ဖြစ်နေပါတယ်။ Main axis ဟာ flex items တွေ laid out ဖြစ်နေတဲ့ primary axis တစ်လျှောက်ပဲဖြစ်ပါတယ်။ အဲ့ဒီ main axis ဟာ Flex container ထဲက flex items တွေရဲ့ direction ကိုသတ်မှတ်ပေးပါတယ်။ ပြီးတော့ cross axis ဟာ main axis နဲ့ perpendicular (ထောင်လိုက်) ဖြစ်နေပါတယ်။ Axis တစ်ခုချင်းစီဟာ start side နဲ့ end side တစ်ခုစီရှိပါတယ်။ Main axis ရဲ့ default ဖြစ်တဲ့ main start နဲ့ main end ရဲ့ direction ဟာ left to right ဖြစ်ပြီး၊ cross axis ရဲ့ default direction ကတော့ top to bottom ပဲဖြစ်ပါတယ်။ ကျွန်တော်တို့ဟာ Flex direction တွေကို အလွယ်တကူ change လုပ်လို့ရပါတယ်။

Flexbox Display Property

အခု ကျွန်တော်တို့ Flexbox ရဲ့ basic concepts နဲ့ terminology တွေကိုသိရှိပြီးတဲ့နောက် Flex container ကိုသတ်မှတ်ပြီး Flexbox layout တစ်ခု create လုပ်လို့ရပါပြီ။ ဒါကြောင့် ကျွန်တော်တို့ရဲ့ html file ထဲမှာ container ဆိုတဲ့ class တစ်ခုနဲ့ div တစ်ခုတည်ဆောက်ပါမယ်။ သူ့ထဲမှာ nested ဖြစ်နေတဲ့ div ခြောက်ခုကို class name ကို item လို့ပေးလိုက်ပါမယ်။ ကျွန်တော်တို့ဟာ flexbox property ကိုမသုံးခင် ကျွန်တော်တို့ layout မှာရှိတဲ့ flex container တစ်ခုကို define လုပ်ဖို့လိုပါတယ်။ ဒါကြောင့် ကျွန်တော်တို့က parent container ဖြစ်တဲ့ div ကို flex container ဖြစ်အောင်လုပ်ပါမယ်။ ဒီလိုလုပ်ဖို့ဆိုရင် ကျွန်တော်တို့ div ထဲက class name container ကို target လုပ်ပြီးတော့ display: flex; လို့ပေးရပါမယ်။ ဒါဆိုရင်တော့ container ဆိုတဲ့ div ထဲမှာရှိတဲ့ item တိုင်းဟာ flex item တွေဖြစ်သွားမှာဖြစ်ပါတယ်။ Browser ကို run ကြည့်တဲ့အခါမှာ flexbox ရဲ့ default အတိုင်း flex items တွေဟာ horizontally အရ main axis ပေါ်မှာ left to right ပေါ်နေမှာဖြစ်ပါတယ်။ ပြီးတော့ cross axis မှာဆိုရင် flex container ရဲ့ top ကနေစပြီး bottom မှာ laid out ဖြစ်နေပါတယ်။ Default အားဖြင့်ဆိုရင် display: flex; လို့ပေးလိုက်ခြင်းဟာ flex container ကို block level တစ်ခုအဖြစ် create လိုက်တာဖြစ်ပါတယ်။ ကျွန်တော်တို့ flex container ဟာ full line ယူတယ်ဆိုပေမယ့် ကျွန်တော်တို့ မယူချင်ရင်လည်းရပါတယ်။ Display value ကို inline-flex လို့ပေးလိုက်မယ်ဆိုရင် flex container ဟာ သူ့ထဲက flex item ရှိသလောက်ရဲ့ space ကိုပဲ ယူပေးတာဖြစ်ပါတယ်။ ကျွန်တော်တို့ မှတ်ထားရမှာကတော့ Flexbox မှာဆိုရင် flex container ရဲ့ အပြင်ဘက်က ဘယ် element ကိုမဆို affect မဖြစ်ပါဘူး။ Flex container တစ်ခုရဲ့ children တွေသာလျှင် flex itemတွေဖြစ်သွားတာပါ။ ကျွန်တော်တို့ နောက်သင်ခန်းစာတွေမှာတော့ Layout direction, order, alignment, sizing နဲ့ နောက်ထပ်အရာတွေအတွက် advanced ဖြစ်တဲ့ Flexbox properties တွေကို ဆက်ပြီးလေ့လာသွားမှာ ဖြစ်ပါတယ်။

Flexbox Direction Property

ပြီးခဲ့တဲ့ သင်ခန်းစာမှာ ကျွန်တော်တို့အနေနဲ့ element တစ်ခုကို display value ဖြစ်တဲ့ flex or inline-flex ပေးပြီး flex container တစ်ခုပြုလုပ်ခဲ့ပြီးပါပြီ။ ကျွန်တော်တို့ဟာ flex container တစ်ခုကို direction order, alignment နဲ့ sizing properties တွေသတ်မှတ်လို့ရပါတယ်။ အချို့သော flexbox properties တွေဟာ flex container ကိုပဲ apply လုပ်ပြီးတော့ အချို့သော flexbox properties တွေကတော့ flex items တွေကိုပဲ apply လုပ်ပါတယ်။ ဒါကြောင့်မို့လို့ ဘယ် properties တွေက flex containers ကို apply လုပ်ပြီး ဘယ် properties တွေက flex items တွေကို apply လုပ်တယ်ဆိုတာကို သိထားဖို့လိုပါတယ်။ ပထမဆုံးတစ်ခုကတော့ flexbox layout ရဲ့ direction ကို control လုပ်ဖို့အတွက်သုံးတဲ့ flex-direction ဆိုတဲ့ property ဖြစ်ပြီး သူဟာ flex items တွေကို affect ဖြစ်မှာမဟုတ်ဘဲ flex container ကိုပဲ affect ဖြစ်မှာဖြစ်ပါတယ်။ ကျွန်တော်တို့ရဲ့ flex direction ကို reverse လုပ်ချင်တယ်ဆိုရင်တော့ flex-direction: row-reverse; လို့ရေးပေးရပါမယ်။ အဲ့ဒီ row-reverse ဟာ main axis ရဲ့ direction ကိုပြောင်းပြန်လုပ်ပေးတာဖြစ်ပါတယ်။ အခုဆိုရင်တော့ ကျွန်တော်တို့ flex container ဟာ right ကနေ left ဘက်ကို reverse လုပ်သွားမှာဖြစ်ပါတယ်။ Flex direction ကို vertical column layout အဖြစ်လိုချင်ရင်တော့ flex-direction ရဲ့ value ကို column လို့ပေးရမှာဖြစ်ပါတယ်။ row-reverse ဆိုတဲ့ value လိုမျိုးပဲ flex direction column ကို reverse လုပ်ချင်တဲ့အခါ flex-direction: column-reverse; လို့သုံးရပါတယ်။

Flex container ထဲမှာရှိတဲ့ flex items တွေဟာ Flex line လို့ခေါ်တဲ့ single line တစ်ခုပေါ်မှာတည်ရှိပါတယ်။ Flex container ဟာ flex line ပေါ်မှာရှိတဲ့ flex items တွေအားလုံးကို သူ့ရဲ့ contents တွေက overflow ဖြစ်တယ်ဆိုရင်တောင်မှာ fix ဖြစ်ဖို့ခက်ခဲပါတယ်။ ဒါကို ကျွန်တော်တို့ browser ကို resize လုပ်ကြည့်တဲ့အခါ flex container ရဲ့ boundaries ကို overflow ဖြစ်သွားပြီး flex items တွေကို wrap မလုပ်တော့ပါဘူး။ ဒါပေမယ့် Flex container ဟာ flex items တွေကို multiple flex lines တွေအဖြစ် လိုအပ်သလို break လုပ်လို့ရပါတယ်။ flex-wrap ဆိုတဲ့ property နဲ့ ကျွန်တော်တို့ flex container ကို single line (သို့) multi-line layout ဖြစ်အောင် control လုပ်လို့ရပါတယ်။ flex-wrap ရဲ့ value ကတော့ wrap ပဲဖြစ်ပါတယ်။ ကျွန်တော်တို့ flex container ထဲမှာ flex-wrap ရဲ့ value ကို wrap ပေးလိုက်ပြီး ကျွန်တော်တို့ browser ကို resize လုပ်ကြည့်တဲ့အခါ ကျွန်တော်တို့ရဲ့ flex items တွေဟာ flex container အတွင်းမှာပဲ သူ့ရဲ့ space ပြည့်သွားတဲ့အခါ multi-line အဖြစ် အလိုလို အောက်ဆင်းသွားမှာဖြစ်ပါတယ်။ အောက်မှာ flex-direction ကို column ပေးပြီး flex items တွေဘယ်လိုနေရာယူသွားလဲဆိုတာ ပြထားပါတယ်။

Creating Sticky Footer

Sticker footer ဆိုတာကတော့ page ရဲ့ content က မည်သို့ပင် ရှိနေပါစေ page ရဲ့အောက်ခြေမှာပဲ stick ဖြစ်နေတာဖြစ်ပါတယ်။ တစ်ကယ်လို့ page ရဲ့ content က browser ရဲ့ height ထက် ပိုတိုနေမယ်ဆိုရင် footer ဟာ bottom နားကို ကပ်မနေဘဲ page ရဲ့ အလယ် content နားမှာပဲ ကပ်နေပါလိမ့်မယ်။ အဲ့ဒီလိုဖြစ်တာကြောင့်ပဲ ကျွန်တော်တို့အနေနဲ့ တစ်ခါတစ်ရံမှာ footer ထဲက view port ရဲ့ bottom ကြားက design လုပ်မထားတဲ့ ကွာဟာမှုတစ်ခုရှိနိုင်ပါတယ်။ ဒါဟာ web design မှာ ဖြစ်လေ့ရှိတဲ့ problem တစ်ခုဖြစ်ပါတယ်။ ကျွန်တော်တို့ဟာ CSS ကိုသုံးပြီး sticky footers တွေ create လုပ်တဲ့အခါမှာ နည်းလမ်းတွေအများကြီးရှိပါတယ်။ CSS layout basic မှာကျွန်တော်တို့က popular method တစ်ခုဖြစ်တဲ့ CSS calc function အသုံးပြုပုံကို လေ့လာခဲ့ပီးဖြစ်ပါတယ်။ ဒါပေမယ့် Flexbox flexibility ကတော့ ဒီလို problems တွေကို ဖြေရှင်းဖို့အတွက် အကောင်းဆုံးနည်းလမ်းတစ်ခုဖြစ်ပါတယ်။ အခြား sticky footer methods တွေမှာဆိုရင် browser ရဲ့ total height ထဲကနေ footer ရဲ့ height ကို တွက်ပြီးတော့ နှုတ်နေရပါသေးတယ်။ ဒါမှမဟုတ် ကျွန်တော်တို့ footer ကို page ရဲ့ bottom မှာပေါ်ဖို့အတွက် div တွေနဲ့ အုပ်နေရပါသေးတယ်။ Flexbox နဲ့ဆိုရင်တော့ calculations တွေ ၊ extra elements တွေထည့်စရာမလိုဘဲ ပြုလုပ်နိုင်ပါတယ်။min-height: 100vh; ဆိုတာ 100% နဲ့ ညီမျှပါတယ်။ အောက်က example မှာ လေ့လာကြည့်ရအောင်။

Beyond The Basics

အခုကျွန်တော်တို့သင်ခန်းစာမှာဆိုရင် basic selector တွေရဲ့ concepts ကိုလေ့လာသွားမှာဖြစ်ပါတယ်။ CSS Basics မှာတုန်းကဆို ကျွန်တော်တို့ဟာ base selectors တွေဖြစ်တဲ့ type, id, class နဲ့ descendant selectors တွေကို လေ့လာခဲ့ပြီးဖြစ်ပါတယ်။ အခုသင်ခန်းစာမှာတော့ CSS selectors တွေကို detail ကျကျလေ့လာသွားပါမယ်။ Selectors တွေဟာ designers တွေနဲ့ developers တွေအတွက် flexibility တစ်ခုဖြစ်ပါတယ်။ ဒါကြောင့် ကျွန်တော်တို့သိထားသလိုပဲ selectors တွေဟာ တစ်ကယ်ကို CSS ရဲ့ important အဖြစ်ဆုံးနဲ့ powerful အဖြစ်ဆုံးတွေဖြစ်ပါတယ်။ ဘာဖြစ်လို့လဲဆိုရင် selectors တွေဟာ ကျွန်တော်တို့ HTML elements တွေကို target လုပ်ရတာဖြစ်ပြီး သူတို့ကို style တွေ apply လုပ်ရတာကြောင့်ဖြစ်ပါတယ်။ နောက်ထပ် attribute selectors, child selectors selector combinators တွေနဲ့ တစ်ကယ်အထောက်အကူဖြစ်တဲ့ pseudo-classes တွေအကြောင်းကို ဆက်လေ့လာသွားမှာဖြစ်ပါတယ်။ Combination of elements တွေကို target လုပ်ဖို့အတွက် ပိုပြီး advanced ဖြစ်တဲ့ selectors တွေနဲ့ simple expressions တွေကိုအသုံးပြုနိုင်ပါတယ်။ ကျွန်တော်တို့ဟာ CSS နဲ့ပဲ ကျွန်တော်တို့ရဲ့ page ထဲကို virtual elements တွေနဲ့ content တွေထည့်ဖို့အတွက် pseudo-elements လို့ခေါ်တဲ့ special selectors တွေကိုတောင်သုံးနိုင်ပါတယ်။

Selector with Attributes

ပြီးခဲ့တဲ့ သင်ခန်းစာတွေမှာလေ့လာခဲ့သလို HTML elements တွေဟာ attributes တွေရဲ့ types အမျိုးမျိုးကို accept လုပ်နိုင်ပါတယ်။ ဥပမာ id attribute, class attribute (သို့) href နဲ့ title attribute တွေဖြစ်ကြပါတယ်။ ဒါကြောင့် CSS ဟာ သူ့ရဲ့ attributes နဲ့ values တွေတဲ့ element တစ်ခုကို target လုပ်တဲ့အခါမှာ flexibility ရှိပါတယ်။ ဒါတွေဟာ form elements တွေကို styling လုပ်ဖို့ တစ်ကယ်ကိုအသုံးဝင်ပါတယ်။ ဥပမာအားဖြင့် ကျွန်တော်တို့ရဲ့ HTML tag ထဲမှာ type attribute, for attribute, placeholder atrribute နဲ့အခြားသော attribute တွေကို သူတို့ရဲ့ attributes နဲ့ values တွေပေါ်မူတည်ပြီး target လုပ်နိုင်ပါတယ်။ ဒါကြောင့် ကျွန်တော်တို့က attribute selector ကို element class or ID name ကို define လုပ်တဲ့အခါ selector တွေက attributes တွေကို match လုပ်ပေးပါတယ်။ ဒါကြောင့် basic attribute selector အများစုဟာ သူ့ရဲ့ value က ဘာပဲဖြစ်နေနေ element attribute တစ်ခုကို target လုပ်ပါတယ်။ Attribute selector ကိုရေးမယ်ဆိုရင်တော့ square bracket ထဲမှာ class လို့ရေးပြီး သူ့နောက်က curly braces ထဲမှာ styling တွေထည့်ရမှာဖြစ်ပါတယ်။

For Example:

        
            form[class="form-contact"] {
                padding: 20px 24px;
                background: #f4f5f8;
            }

            div[id="container"] {
                max-width: 500px;
                margin: auto;
            }

            input[type="text"] {
                background: #fdfee6;
            }

Styling Mouser Hover & Anchor Elements

အခုသင်ခန်းစာမှာ ကျွန်တော်တို့က form ထဲက buttons တွေကို target လုပ်ပြီး mouse cursor အသစ်တစ်ခု define လုပ်မှာဖြစ်ပါတယ်။ ကျွန်တော်တို့ရဲ့ button ကို hover လုပ်တဲ့အခါမှာ pointer cursor ကိုပြချင်တဲ့အတွက် CSS file ထဲမှာ button, reset နဲ့ submit တို့ကို target လုပ်ပြီး သူ့ရဲ့ curly braces {} အတွင်းထဲမှာ cursor: pointer; လို့ပေးသတ်မှတ်ပေးရပါမယ်။ ပြီးရင် browser ကိုသွားပြီး button တွေကို hover လုပ်လိုက်တဲ့အခါ pointer style က ပြောင်းသွားမှာဖြစ်ပါတယ်။ နောက်ဆုံးမှာ ကျွန်တော်တို့ဟာ page ပေါ်မှာ class or id names တွေတူတဲ့ elements တွေရှိနိုင်ပါတယ်။ ဒါပေမယ့် သူတို့ထဲက တစ်ချို့မှာလည်း special attribute တွေ assign လုပ်ထားတာရှိနိုင်ပါတယ်။ ဥပမာ link လိုမျိုး anchor elements တွေပေါ့။ သူတို့ကို target="_blank"; လို့ပေးလိုက်တာဟာ အဲ့ဒီ link ကိုနှိပ်လိုက်တဲ့အချိန် browser ကနေ new window ကို open လုပ်ပေးတာဖြစ်ပါတယ်။ သူတို့ကို CSS ထဲမှာ a[target="_blank"]{ } လို့ရေးပြီး style တွေဖန်တီးနိုင်ပါတယ်။ ဒါကို အောက်က example မှာကြည့်ရအောင်။

For Example:

        
            input[type="button"],
            input[type="reset"],
            input[type="submit"] {
                cursor: pointer;
            }


            a[target="_blank"] {
                color: #39add1;
                text-decoration: none;
                border-bottom: 1px dotted;
            }

DRY (Don't Repeat Yourself)

ဒီတစ်ခါမှာတော့ ကျွန်တော်တို့ style declarations တွေမထပ်အောင်ရေးတဲ့နည်းလေးကိုပြောပြချင်ပါတယ်၊ သူ့ကို web development မှာဆိုရင် DRY (don't repeating yourself) လို့ခေါ်ပါတယ်။ သူ့ရဲ့ ရည်ရွယ်ချက်ကတော့ same bits of code တွေကို ထပ်ခါထပ်ခါ ရေးနေရတာတွေကို ရှောင်စေချင်တာပဲဖြစ်ပါတယ်။ ကျွန်တော်တို့ css file ထဲမှာ တူညီနေတဲ့ code တွေထပ်ခါထပ်ခါ ထပ်နေရင် ကျွန်တော်တို့ code တွေကိုပြင်ချင်တဲ့အခါ တူတဲ့နေရာတိုင်းကိုလိုက်ပြင်ရလို့အချိန်ကြာတာရယ်၊ ကျွန်တော်တို့ရဲ့ code ကို execute လုပ်တဲ့အချိန်မှာလဲ run time ကြာပီးတော့ ကျွ်တော်တို့ရဲ့ web site ကလဲ မြန်မြန်ဆန်ဆန်တတ်မလာပါဘူး။ ဒါကြောင့် အဲ့လိုဟာတွေကိုရှောင်ချင်တယ်ဆိုရင်ကျွန်တောတို့ DRY ကိုသုံးဖို့လိုပါတယ်။ ဥပမာ ကျွန်တော်တို့က p tag တွေမှာရော h tag တွေမှာရော background color ကိုတစ်ခုတည်းထည့်ချင်တယ်ဆိုရင်တော့ တစ်ခုချင်းဆီကို background color လိုက်ထည့်နေမဲ့အစား ကျွန်တော်တို့ bgcolor ဆိုတဲ့ class ကို p tag နဲ့ h tag တွေမှာထည့်ပီးတော့ ကျွန်တော်တို့ရဲ့ css property ကိုတစ်ခုတည်းမှာရေးပါတယ်၊ အဲ့လိုမျိုးရေးလိုက်ချင်းကြောင့် ကျွန်တော်တို့က color ကိုလိုက်ပြင်ချင်တယ်ဆိုရင်တောင် တစ်ခုတည်းပြင်လိုက်ရုံနဲ့ အကုန်လုံးပြင်ပီးသွားဖြစ်သွားမှာပါ။ ပီးတော့ ကျွန်တော်တို့ရဲ့ code ကလဲ 10 ကြောင်းလောက်ရေးရမှာကို တစ်ကြောင်းတည်းနဲ့ ရေးလို့ရသွားတဲ့ အတွက် execute လုပ်တာလဲမြန်သွားမှာပါ။ ဒါကြောင့်ကျွန်တော်တို့ code တွေတတ်နိုင်သမျှမထပ်အောင်ပြုလုပ်ဖို့လိုပါတယ်၊ အမြဲတမ်းသုံးနေတဲ့ကောင်တွေကို class name သီးသန့်ပေးထားပီးတော့ လိုတဲ့အချိန်တိုင်းသူ့ကိုခေါ်သုံးတာကောင်းပါတယ်။

Combinators

Combinator ဆိုတာ selector တစ်ခုအတွင်းမှာရှိတဲ့ elements တွေကြားက relationship ကိုခေါ်ပါတယ်။ ဒါကြောင့် ကျွန်တော်တို့ဟာ နှစ်ခု (သို့) နှစ်ခုထက်ပိုတဲ့ elements တွေကို selector တစ်ခုနဲ့ target လုပ်မယ်ဆိုရင် combinator ကိုသုံးဖို့လိုပါလိမ့်မယ်။ ကျွန်တော်တို့ ရှေ့ပိုင်းမှာအသုံးပြုခဲ့တဲ့ descendent selectors တွေကလည်း combinator တစ်ခုပဲဖြစ်တယ်။ ဘာလို့လဲဆိုရင် descendent selector ဟာ selector နှစ်ခုကြားမှာ space ခြားထားတာရယ်၊ descendent elements တွေကို target လုပ်နိုင်လို့ပဲဖြစ်ပါတယ်။ Combinators တွေဟာ element တစ်ခုရဲ့ ဘယ်လို sibling ကိုမဆို ပြောင်းလဲလို့လွယ်အောင် ပြုလုပ်ပေးထားပါတယ်။ Combinators တွေကို symbol အဖြစ် greater than ( >) sign, plus (+) sign နဲ့ tilde symbol ဆိုပြီး ကျွန်တော်တို့ရဲ့ selectors တွေကို ပိုပြီးတိကျစွာ target လုပ်ဖို့အတွက် သုံးနိုင်ပါတယ်။ Greater than sign ကို child combinator အဖြစ်အသုံးပြုပါတယ်။ Child combinators တွေကိုသုံးတဲ့ child selectors တွေလို့လည်းခေါ်ပါတယ်။ ဘာလို့လဲဆိုရင် သူတို့ဟာ အခြား elements တွေရဲ့ direct children တွေဖြစ်တဲ့ elements တွေကို target လုပ်နိုင်လို့ပါ။ Direct children တွေဟာ parent element တစ်ခုထဲမှာ nested ဖြစ်နေပါတယ်။ နောက်ထပ်ကျွန်တော်တို့အသုံးပြုနိုင်တဲ့ combinator ကတော့ plus sign ပဲဖြစ်ပါတယ်။ Plus sign combinator ကတော့ ကပ်လျှက်ရှိနေတဲ့ sibling combinator ပဲဖြစ်ပါတယ်။ ဘာဖြစ်လို့လဲဆိုရင် page တစ်ခုမှာရှိတဲ့ next sibling elements တွေကိုပဲ target လုပ်နိုင်လို့ပါ။ ဥပမာ ကျွန်တော်တို့က ကပ်လျှက်ရှိတဲ့ button နှစ်ခုရှိတယ်ဆိုပါစို့။ ဒုတိယ button ရဲ့ style ကိုပဲပြောင်းချင်တယ်ဆိုရင် plus sign combinator ကိုအသုံးပြုနိုင်ပါတယ်။ နောက်ဆုံးတစ်ခုကတော့ tilde symbol (~) ပဲဖြစ်ပါတယ်။ သူ့ကို general sibling combinator လို့လည်းခေါ်ပါတယ်။ ဘာလို့လဲဆိုရင် သူက siblings တွေကို target လုပ်တယ်ဆိုပေမယ့် sibling ကိုပဲ target လုပ်ရုံသာမက တိကျတဲ့ sibling တိုင်းကိုပါ target လုပ်ပေးပါတယ်။ အောက်က ဥပမာအရပြောရရင် h1 ရဲ့ နောက်က label element တိုင်းကို target လုပ်ပေးမှာပဲ ဖြစ်ပါတယ်။
